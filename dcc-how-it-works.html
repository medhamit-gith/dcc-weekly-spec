How the data flows
✅

Simple and reliable

A dedicated "DCC Bot" account fetches the club's cycling data from Strava 3 times a day — at 8am, 1pm, and 8pm. Your app and the web dashboard read from this pre-fetched cache. No individual member needs a Strava account to view the stats.

How it works on each device
Step 1 — First time only
🔑 Enter your club code
Open the app and enter your name and the DCC club access code (shared via the club WhatsApp group). No Strava account needed — the club bot handles all data fetching on your behalf.
The club code verifies you're a DCC member. It's stored on your device so you only enter it once.
Step 2 — Every time you open the app
🔐 Face ID / Touch ID
The app uses Face ID (or Touch ID on older iPhones) to verify it's you. No passwords. Even if someone picks up your phone, they can't access the app without your face or fingerprint.
The biometric check happens every time. There's no grace period — it always verifies.
Step 3 — Automatic
⚡ Club data loads from Cloudflare
The app fetches this week's club activities from a Cloudflare server — NOT from Strava directly. The DCC Bot has already collected the data 3 times today and cached it for everyone.
12 members opening the app = 12 reads from the cache (unlimited), instead of 12 direct calls to Strava.
Step 4 — You're in
📊 See your club's stats
Charts, tables, leaderboards, rider analysis, insights, coaching tips — all displayed instantly. Even without internet, the app shows the last data it downloaded.
Five tabs: Overview, Leaderboard, Charts, Insights, and Analysis — with drill-down into individual rider stats.
Step 1 — Open the link
🌐 Open the dashboard in any browser
Open the club dashboard link (shared via WhatsApp) in Chrome, Safari, or any browser. No app to install — it just works on any phone, tablet, or computer.
Link: medhamit-gith.github.io/dcc-weekly-spec/dashboard.html
Step 2 — You're in
📊 Same data, any device
The web dashboard shows the same leaderboard, data table, and activity feed as the iOS app. Same data, same update frequency (3x daily). Bookmark it or add to your home screen.
Auto-refreshes every 5 minutes. No login required.
Step 3 — Optional
📌 Add to home screen
On Android: browser menu → "Add to Home screen". On iPhone Safari: share button → "Add to Home Screen". It looks and feels like a native app.
Step 1 — First time only
📺 Enter club code on TV
Enter your name and "DCC2026" using the Siri Remote's on-screen keyboard. Same club code as the iPhone app.
One-time setup — the code is saved on the Apple TV.
Step 2 — Automatic
⚡ Same data, TV-optimised
Reads from the same Cloudflare cache. Designed for the living room — big numbers, bold text, Siri Remote navigation.
Step 3 — You're in
📊 Big screen leaderboard
Perfect for the clubhouse. Navigate between Overview, Leaderboard, Insights, and Analysis tabs with the Siri Remote. Parallax effects on rider cards give it that premium Apple TV feel.
Your data is protected
🔑 Club code access
The iOS app requires a shared club code to access. No Strava account needed. The code is shared only with DCC members via WhatsApp.
🔐 Face ID / Touch ID
Verifies your identity with biometrics every time you open the app. Uses Apple's Secure Enclave hardware — your biometric data never leaves the device.
🔒 HTTPS everywhere
Every connection uses encrypted HTTPS (TLS 1.3). No data travels unencrypted.
👤 What the app can see
Only aggregate club stats: first name + last initial, distances, speeds, elevation, ride titles. CANNOT see GPS routes, heart rate, power, private activities, or email addresses.
🚫 What the app cannot do
Cannot post on your behalf, modify activities, message athletes, track your location, or access anything beyond public club statistics.
🤖 The DCC Bot
A dedicated Strava account controlled by the app administrator. Fetches club data 3x daily using read-only permissions. Your personal Strava account is never touched.
🔐 Secret isolation
The Strava API key lives ONLY on the Cloudflare server — never in the app. Even if someone decompiled the app, they wouldn't find it.
📋 Privacy compliance
The app declares no tracking, no tracking domains, no third-party analytics via Apple's PrivacyInfo.xcprivacy manifest.
🛡️

No personal data leaves your device

The only data that travels over the internet is aggregate club stats (distances, speeds, elevation) — the same information visible on Strava's club page. No GPS routes, no heart rate, no email addresses, no athlete IDs. Strava strips all of this from the club data endpoint for privacy.

Jargon buster
API
The way apps talk to each other. When DCC fetches data from Strava, it uses Strava's API — a structured way to request information.
Cloudflare Worker
A small program running on Cloudflare's global servers. It fetches club data from Strava, caches it, and serves it to all DCC members. A fast middleman.
Rate limit
A cap on how many times an app can ask Strava for data. The Bot architecture avoids this by making only ~9 requests per day total.
TestFlight
Apple's official app for testing apps before they go to the App Store. Only invited people can install. Free and safe.
Face ID
Apple's facial recognition. Uses the iPhone's TrueDepth camera to create a 3D map of your face. Data stays in the Secure Enclave chip — never leaves your device.
HTTPS
The encrypted version of HTTP. The padlock in your browser means HTTPS is active. All DCC data travels over HTTPS — nobody can eavesdrop.
